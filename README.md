# AK firmware custom porting - STM32F103C8T6

This repository is forking from the [AK-base-kit STM32L151](https://github.com/ak-embedded-software/ak-base-kit-stm32l151) repository with the aim of modifying the source code to provide additional support for the specific MCU family, namely STM32F103C8T6.

## Project Structure

The project structure is organized as follows:

- `stm32f_porting`: This directory contains the modified source code for the STM32F103C8T6 MCU. It includes necessary changes to accommodate the differences between the STM32L151 and specific STM32F103C8T6, such as peripheral configurations, clock settings, and pin mappings. The platform code is auto-generated by STM32CubeMX, which provides a convenient way to set up the project for the STM32F103C8T6 without rewriting the entire codebase from scratch.
- `stm32l_base`: This directory contains the original source code for the STM32L151 MCU. It serves as a reference for the modifications made in the `stm32f_porting` directory. The code in this directory is not modified and is kept intact to maintain a clear distinction between the original and modified codebases. This directory is also contain the original platform code for AK firmware, which is used as a reference for the modifications made in the `stm32f_porting` directory.
- `README.md`: This file provides an overview of the project, including its purpose, structure, and instructions for use.
- `.gitignore`: This file specifies which files and directories should be ignored by Git, such as build artifacts, temporary files, and other non-essential files that do not need to be tracked in version control.
- `LICENSE`: This file contains the licensing information for the project, specifying the terms under which the code can be used, modified, and distributed.

## Usage Instructions

At the moment, the project is focused on the development and modification of the source code for the STM32F103C8T6 MCU. The modified code in the `stm32f_porting` directory is intended to be used as a reference for developers who want to port the AK firmware to the STM32F103C8T6 platform.

Therefore, there are no specific usage instructions to run the code at this stage, as the project is still in the development phase. However, developers can refer to the modified code in the `stm32f_porting` directory to understand the changes made to accommodate the STM32F103C8T6 MCU and use it as a basis for their own porting efforts.

## Note

Please note that the modifications made in the `stm32f_porting` directory are specific to the STM32F103C8T6 MCU and may not be compatible with other MCU families. It is important to review the changes carefully and ensure that they are appropriate for your specific use case before using or modifying the code further.

Additionally, the base project is covered and worked on Linux environment, so it is recommended to use a Linux-based development environment for working with this project to ensure compatibility and ease of development.

If you are using a different operating system, please ensure that you have the necessary tools and dependencies installed to work with the STM32F103C8T6 MCU and the AK firmware.

For example, with Windows, WSL (Windows Subsystem for Linux) can be used to provide a Linux-like environment for development, allowing you to work with the project without needing to set up a separate Linux machine.
